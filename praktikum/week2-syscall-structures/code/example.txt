brk() / mmap() Manajemen memori, Mengatur heap dan memetakan library ke memori.

2. Eksperimen 2 - menelusuri system call file I/O
strace -e trace=open,read,write,close cat /etc/passwd
analisis bagaimana ile di buka,di baca dan di tutup oleh kernel
- membuka file 
strace -e trace=open 
Kernel mencari file /etc/passwd di sistem file. Jika ditemukan dan pengguna punya izin baca, kernel membuat
ile descriptor (FD), misalnya 3, yang menjadi “pegangan” program ke file itu.
- membaca isi file
read(3, buffer,
rogram cat meminta kernel membaca hingga 4096 byte dari file descriptor 3. Kernel membaca blok data dari disk
(melalui page cache) dan menyalinnya ke buffer cat. Hasilnya dikembalikan ke user space.
- menulis ke layar
write
Program cat lalu mengirim isi buffer ke file descriptor 1, yaitu stdout (layar terminal).Kernel meneruskan data ini ke
driver terminal agar muncul di layar.
- menutup file
close cat /etc/passwd
Setelah selesai membaca, cat meminta kernel menutup file descriptor 3.Kernel melepaskan semua struktur
internal yang terkait dengan file tersebut.

3. Eksperimen 3- mode user vs kernel
dmesg | tail -n 10
amati log kernel yang muncul, apa perbedaan output ini dengan output program biasa
- Output program biasa → dihasilkan oleh proses user-space, lewat system call write() ke stdout (layar).
Contoh: ls, cat, echo, python, dll.
- Log kernel (dmesg) → dihasilkan oleh kode kernel (C di dalam Linux), dan dikirim ke ring buffer kernel, bukan langsung ke layar.
Hanya ditampilkan lewat dmesg atau terbaca lewat /proc/kmsg.

3. a. mengapa system call sangat penting untuk keamana os?
  -System call merupakan komponen penting dalam menjaga keamanan sistem operasi karena berfungsi sebagai penghubung antara ruang pengguna (user space) dan ruang kernel (kernel space). Semua akses ke sumber daya penting seperti file, memori, dan perangkat keras harus melalui system call, sehingga kernel dapat mengontrol setiap tindakan yang dilakukan program. Dengan adanya mekanisme ini, sistem operasi dapat memeriksa izin akses, mencegah eksekusi kode berbahaya, dan membatasi operasi yang tidak sah. System call juga memungkinkan penerapan kebijakan keamanan seperti sandboxing dan access control list, yang menjaga agar proses pengguna tidak dapat mengganggu proses lain atau sistem inti. Selain itu, aktivitas system call dapat dicatat untuk keperluan audit dan deteksi serangan. Tanpa system call, program dapat langsung mengakses kernel, yang akan sangat berisiko terhadap integritas, stabilitas, dan keamanan sistem operasi secara keseluruhan.
 b. Bagaimana OS memastikan transisi user-kernel berjalan aman?
  -OS memastikan transisi dari user mode ke kernel mode berjalan aman dengan cara membatasi jalur masuk (system call), memvalidasi setiap parameter, dan memanfaatkan proteksi hardware CPU.
Dengan kombinasi mekanisme perangkat keras dan perangkat lunak, OS mencegah program user memodifikasi atau mengakses area kernel secara langsung.
